# pyfovea

> **Standing test gate** (run from the repo root; the venv is created once
> with `/opt/homebrew/bin/python3 -m venv pyfovea/.venv` and
> `pyfovea/.venv/bin/pip install -e "pyfovea[dev]"`):
>
> ```sh
> pyfovea/.venv/bin/python -m pytest pyfovea/tests
> ```

Python reader and CLI for **FoveaCam Duo** recordings. Supports both
on-disk formats:

- **`.fovea`** — the single-file container written by the app's recorder
  (standard [MCAP](https://mcap.dev) inside; schema contract:
  `docs/refactor/recorder-container.md` §2b). Includes a **streaming
  recovery path** for crash-truncated (footerless) files: every complete
  record before the truncation point is recovered.
- **legacy `.stream`/`.meta` dumps** — the per-stream blob + JSONL sidecar
  format older recordings use. This package absorbs the retired per-dump
  `stream-decoder.py` template's decode logic (12-bit significant-bits
  scaling, affine extras); old dumps stay loadable forever.

> The package name `pyfovea` is a placeholder; it may be renamed before any
> PyPI release. **Publishing is user-gated** — this repo only prepares the
> packaging.

## API

```python
from pyfovea import FoveaReader, LegacyRecording, convert_legacy

with FoveaReader("dump/recording.fovea") as r:
    r.streams                       # {name: StreamInfo(dtype, shape, pixelFormat, …)}
    r.session                       # {"timestamp": …, "durationSec": …}
    r.truncated                     # True → recovered via streaming scan
    for frame in r.iter_frames("left-fovea"):
        frame.raw                   # np.ndarray — 12p payloads auto-unpacked to uint16
        frame.display               # uint8, scaled by TRUE bit depth (12-bit → /4095)
        frame.t, frame.seq          # seconds, per-stream sequence
        frame.extra                 # telemetry: volt / angle / affine …
        frame.affine                # 3×3 np.ndarray or None

with LegacyRecording("old-dump-dir/") as rec:
    rec.manifest                    # manifest.json contents
    rec.streams["left-fovea"][0].raw

convert_legacy("old-dump-dir/", "old-dump.fovea")  # legacy → .fovea (§2b schema)
```

Decoding details live in `pyfovea.dtypes`: `unpack_12p` (GenICam `*12p`,
3 bytes → 2 pixels), `decode_payload` (handles packed and unpacked storage),
`to_display` (significant-bits scaling), `demosaic` (optional, needs the
`cv` extra — `pip install pyfovea[cv]`).

## CLI

```sh
pyfovea inspect  recording.fovea          # streams, counts, timing, session meta
pyfovea inspect  old-dump-dir/            # same summary for a legacy dump
pyfovea export   recording.fovea -o out/  # frames → .npy (+ .json extras)
pyfovea export   recording.fovea -f pgm   # 8-bit display-scaled .pgm (mono)
pyfovea convert  old-dump-dir/ -o out.fovea
```

`inspect` works on crash-truncated files (flags them as `[TRUNCATED]`).

## Tests & fixtures

`tests/fixtures/` is checked in and was generated by the **real app
recorder** (B-5 harness) — see `tests/fixtures/generate.ts` for the exact
script and regeneration instructions. `tiny.fovea` carries a 12p-packed
stream with telemetry extras and a Mono16 stream; `crash-truncated.fovea`
is a chunk-per-message recording cut at 60% to exercise the recovery path;
`legacy/` is a real legacy dump.
