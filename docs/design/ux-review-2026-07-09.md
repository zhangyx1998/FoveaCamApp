## Bundle B — calibration workflow guidance

The manual pass confirmed the prerequisite chain (center intrinsic →
extrinsic → full triple for distortion/drift) is enforced in code but taught
nowhere in the UI.

- **B1 (major) Calibration order is undiscoverable.** The four apps are
  launcher peers; nothing shows intrinsic→extrinsic→distortion→drift or
  which steps are done/blocked. Suggest a calibration hub/stepper (or
  done/blocked annotations on the launcher entries) derived from what
  calibration data exists. (calibration.ts gating; no ordering in any UI.)
- **B2 ✔ (major) Missing-prerequisite failure is mute/misdirecting.**
  Planner-verified mechanism: `loadConversions`' "No extrinsic data for
  regression" throw escapes `acquireTriple`, is caught by
  resource-session.ts:154 and goes out the `report()` channel — the session
  never `fail()`s, so distortion/drift freeze mid-progress-monitor with no
  error banner. (The `TRIPLE_UNAVAILABLE` message fires only on lease
  failure.) Suggest: catch calibration-data errors in `acquireTriple` and
  `fail()` with a message naming the missing step ("Extrinsic calibration
  required — run it first").
- **B3 (major) No collection guidance in intrinsic.** No sample-count
  target, no frame-coverage indication, no per-record reprojection error
  after the solve — bad sets are discovered late and pruned by guesswork.
  (calibrate-intrinsic contract carries only recordCount + lastRms.)
- **B4 (major) Disabled buttons give no reason.** Extrinsic **Capture**
  (controller? which camera unlocked?) and Drift **Update Drift** ("no
  lock" vs "no change" — opposite operator actions, same grey) need a
  one-line disabled-reason. One shared pattern serves both.

## Bundle C — recording/capture visibility

- **C1 (major) Recording health is hover-only.** The per-stream stats table
  is `v-if="isRecording && hover"` + `pointer-events:none` — the only live
  drop readout is hidden unless the cursor camps on the title-bar button.
  Suggest an at-distance persistent cue (tint the blinking record dot when
  drops > 0), hover table stays for detail. (RecordButton.vue:83,190)
- **C2 (major) `q`/`r` drop codes and `pub`/`wrote` headers are
  developer jargon** with no legend/tooltip anywhere in the UI. Cheap fix:
  `title=` legend on the drops column ("q = writer behind, r = source
  ahead"). (RecordButton.vue:88-107)
- **C3 (major) Degraded raw-stack captures have no badge** — the only
  signal is a collapsed `capture_meta` JSON row (`capture: "raw-stack"`,
  `wrap: "none"`). Suggest a persistent `--warn` badge on affected preview
  tiles ("RAW — no warp"). (capture-helper.ts:174,208; preview-meta/Item.vue:39)
- **C4 (polish) Shortcuts unadvertised** — `Cmd/Ctrl-R` / `Cmd/Ctrl-S`
  appear in no tooltip or menu (`Cmd/Ctrl-O` is the only advertised one).
  Append accelerators to the button `title=`s. Pairs with the stale
  "Capture preview" tooltip on the camera icon, which now toggles a window
  and shows no open/closed state.
- **C5 (polish) No bit-depth warning on save format** — saving a
  full-depth (12/16-bit) capture as JPG silently downconverts. Annotate the
  dropdown or warn.
- **C6 (polish) Capture shot is a bare unstyled button** in a window where
  it's the primary action (defers to P3 shared Button).

## Bundle D — gesture discoverability + cross-app consistency

- **D1 (major) Drag-to-steer is invisible in all three operator apps** —
  the primary gesture has no cursor change, hint, or affordance. Minimum:
  `cursor: crosshair` on steerable views; better: a faint first-use hint.
  (Concrete instance of P5.)
- **D2 (major) Disparity Scope override drag silently resets the PID** —
  the only explanation is a tooltip on the **override** badge that appears
  after the reset already happened. Suggest a persistent one-line hint near
  the wide view and/or a snap-in "override — PID reset" toast.
- **D3 (major) Three apps, three idioms** — Tracking-Multi's bespoke grid
  vs the shared L/C/R+drawer shell; center-view selector is a `<select>` in
  Manual Control but an in-title `InlineSelect` in Disparity Scope; voltage
  telemetry differs per app. Converge on one shell (ties to P6), one
  selector idiom, both-eye volts everywhere.
- **D4 (minor) Marker-lock ✓/✗ is a tiny monochrome glyph** — the most
  important live signal in extrinsic/distortion/drift. Colorize (green/red),
  reserve fixed width, consider mirroring onto the stream outline. (Ties to
  P7/P10.) Also: the detector-Hz liveness footnote exists only in intrinsic;
  extend it to the triple apps.
- **D5 (minor) Mode is invisible in Manual Control** — set-point selection
  vs free drag is a subtle state machine (slider nudges silently clear the
  selection); show which source is authoritative near the target readout.
- **D6 (polish) Debugger button shows no open/closed state** — give it the
  same `.toggle.active` idiom as the Tracker on/off two columns over.
- **D7 (polish) Zoom inputs inconsistent/unvalidated** — Manual Control's
  is bare text (silently clamped), Disparity Scope's is `type="number"`.

## Bundle E — viewer + shell odds and ends

- **E1 (minor) Viewer "Reset UI state" has no confirm** — the one
  *manual* destructive layout action is the only unguarded one (corrupt/
  mismatch paths do prompt). Route through the existing confirm dialog.
  (ViewerWindow.vue:706 vs :222-244)
- **E2 (minor) `v`-to-disable and click-to-focus are undiscoverable, and
  keyboard-only** — add a small per-block toggle icon and/or a one-line
  header hint.
- **E3 (major) "Single Capture" doesn't capture** — it is a live monitor
  with both title-bar actions permanently dead. Rename ("Live View") or
  give it real single-shot capture. **Needs ruling: rename vs extend.**
- **E4 (minor) `manage-data` is an unreachable empty stub** (zero inbound
  references; three 0-byte views). Delete, or track "in-app data browser"
  as an explicit gap. **Needs ruling: delete vs roadmap.**
- **E5 (minor) Tracking-Multi advertises 8 target slots, renders 4** —
  contract MAX=8 but `state.targets` seeds 4 with no add control. Expose
  8 or pin 4 as the documented max. **Needs ruling.**
- **E6 (minor) Manage Cameras has no Rescan** — discovery runs once on
  mount; the "in use by another process" error can only be cleared by
  reopening the app. Add a **Rescan** button wired to `refresh`.
- **E7 (minor) Welcome launcher holds a live camera stream** just to show
  a thumbnail — the idle launcher is a camera-holding window (relevant to
  the registry-utilization ceiling). Consider a low-rate/on-demand preview.
- **E8 (polish) Welcome camera dropdown mounts conditionally**
  (`v-if list.length > 1`) — layout jump when the count crosses 1↔2;
  reserve the space (ruled principle 3).
- **E9 (polish) Single Capture empty state offers no guidance** when no
  cameras are connected ("No cameras — open Manage Cameras").
- **E10 (minor) Distortion projection freezes stale on marker loss** with
  no cue — a dropped detection masquerades as bad calibration in the one
  tool whose job is to be trusted by eye. Blank or "stale"-label the panel
  on loss.
- **E11 (minor) Intrinsic thumbnails are click-to-delete with only a
  tooltip** — P5 instance; persistent affordance + explicit × on hover.
