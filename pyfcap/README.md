# fcap

> `/opt/homebrew/bin/python3 -m venv pyfcap/.venv` and
> `pyfcap/.venv/bin/pip install -e "pyfcap[dev]"`):
>
> ```sh
> pyfcap/.venv/bin/python -m pytest pyfcap/tests
> ```

Python reader and CLI for **FoveaCam Duo** recordings. Supports both
on-disk formats:

- **`.fcap`** — the single-file container written by the app's recorder
  (standard [MCAP](https://mcap.dev) inside). Includes a **streaming
  recovery path** for crash-truncated (footerless) files: every complete
  record before the truncation point is recovered. Legacy **`.fovea`**
  files use the identical container and read the same way — the reader is
  extension-agnostic, so older recordings stay loadable.
- **legacy `.stream`/`.meta` dumps** — the per-stream blob + JSONL sidecar
  format older recordings use. This package decodes them directly (12-bit
  significant-bits scaling, affine extras); old dumps stay loadable.

> Distribution and import name: **`fcap`**. Confirm the PyPI name `fcap` is
> available before publishing.

## API

```python
from fcap import FoveaReader, LegacyRecording, convert_legacy

with FoveaReader("dump/recording.fcap") as r:
    r.streams                       # {name: StreamInfo(dtype, shape, pixelFormat, …)}
    r.session                       # {"timestamp": …, "durationSec": …}
    r.truncated                     # True → recovered via streaming scan
    for frame in r.iter_frames("left-fovea"):
        frame.raw                   # np.ndarray — 12p payloads auto-unpacked to uint16
        frame.display               # uint8, scaled by TRUE bit depth (12-bit → /4095)
        frame.t, frame.seq          # seconds, per-stream sequence
        frame.extra                 # telemetry: volt / angle / affine …
        frame.affine                # 3×3 np.ndarray or None

with LegacyRecording("old-dump-dir/") as rec:
    rec.manifest                    # manifest.json contents
    rec.streams["left-fovea"][0].raw

convert_legacy("old-dump-dir/", "old-dump.fcap")  # legacy → .fcap
```

Decoding details live in `fcap.dtypes`: `unpack_12p` (GenICam `*12p`,
3 bytes → 2 pixels), `decode_payload` (handles packed and unpacked storage),
`to_display` (significant-bits scaling), `demosaic` (optional, needs the
`cv` extra — `pip install fcap[cv]`).

## CLI

```sh
fcap inspect  recording.fcap          # streams, counts, timing, session meta
fcap inspect  old-dump-dir/           # same summary for a legacy dump
fcap export   recording.fcap -o out/  # frames → .npy (+ .json extras)
fcap export   recording.fcap -f pgm   # 8-bit display-scaled .pgm (mono)
fcap convert  old-dump-dir/ -o out.fcap
```

`inspect` works on crash-truncated files (flags them as `[TRUNCATED]`) and
on legacy `.fovea` containers.

## Tests & fixtures

`tests/fixtures/` is checked in and was generated by the **real app
recorder** — see `tests/fixtures/generate.ts` for the exact script and
regeneration instructions. The container fixtures keep their
original `.fovea` extension (legacy-named on purpose — they double as the
legacy-extension read-path proof; the reader ignores the suffix).
`tiny.fovea` carries a 12p-packed stream with telemetry extras and a
Mono16 stream; `crash-truncated.fovea` is a chunk-per-message recording
cut at 60% to exercise the recovery path; `legacy/` is a real legacy dump.
